# Walk-forward 稳健性实验：双均线策略有没有可泛化的 edge？

> 复现：`cd example && python run_experiments.py`（数据走 yfinance，无需账号）。
> 实现见 `example/wf.py`（前推评估协议）+ `example/run_experiments.py`（实验台）。

## 动机

原始演示用一次 70/30 切分（单个 OOS 样本）+ 「候选 score > 基线」门槛接受。这套口径
有三个会**误判**的漏洞，前文 12 标的扫描已暴露：

1. 单段样本内挑参，OOS 只有一个点，运气主导。
2. 接受门槛会在**负分**、**3 笔交易**上误报「部署」。
3. 高相关指数（^GSPC/SPY/QQQ…）被当成多个独立确认。

## 升级后的固定评估协议

| 维度 | 原始 | 升级 |
| --- | --- | --- |
| 样本 | 1 次 70/30 | **滚动前推**：train 504d / test 126d / step 126d，每标的 28 个 OOS fold |
| 挑参 | 全段一次 | **adaptive**：每个 train 窗口重选最优配置，应用到下一段未见 test |
| 基准 | 与基线参数比 | 与**买入持有（B&H）**的 OOS Sortino 比 |
| 部署门槛 | score > 基线 | OOS Sortino 中位数 > 0 **且** 均值 > B&H **且** ≥50% fold 跑赢 B&H **且** OOS 总笔数 ≥ 20 |

## 实验清单与结果

数据 2010-01-01 ~ 2026-02-28（BTC 自 2015），候选 `{5/10, 10/20, 10/30, 20/60, 15/40, 30/100}`。

### E1+E2 — 篮子前推（1x 成本）

| 资产 | OOS Sortino 中位/均值 | B&H Sortino | %跑赢B&H | OOS笔数 | 部署 |
| --- | ---: | ---: | ---: | ---: | :--: |
| ^GSPC | +1.06 / +1.20 | +1.85 | 29% | 49 | no |
| ^IXIC | +0.83 / +1.01 | +1.83 | 29% | 67 | no |
| ^DJI | +0.69 / +0.93 | +1.75 | 21% | 71 | no |
| SPY | +0.90 / +1.20 | +2.07 | 21% | 61 | no |
| QQQ | +1.11 / +1.45 | +1.99 | 32% | 56 | no |
| AAPL | +0.32 / +0.75 | +1.98 | 18% | 94 | no |
| MSFT | +0.94 / +0.75 | +1.95 | 18% | 82 | no |
| NVDA | +1.48 / +1.38 | +2.38 | 29% | 78 | no |
| AMZN | +1.33 / +1.19 | +1.82 | 32% | 88 | no |
| GLD | +0.72 / +0.23 | +0.88 | 36% | 102 | no |
| TLT | −0.21 / −0.06 | +0.36 | 43% | 81 | no |
| BTC-USD | +1.37 / +1.45 | +1.85 | 43% | 63 | no |

**严格门槛通过：0/12。** 双均线在每个资产上的 OOS 风险调整收益都**低于直接持有**，
跑赢 B&H 的 fold 比例从未达到一半。择时没有创造超额，反而牺牲了在场时间。

### E3 — 旧弱门槛 vs 新严格门槛（同一份 WFA 结果）

旧门槛部署 **12/12**，新门槛部署 **0/12** → **拦下 12 个假阳性**。
这正是问题所在：策略确实经常产出正的 OOS Sortino，旧口径据此「接受」，但它们都跑输
buy-and-hold。把基准从「基线参数」换成「什么都不做」，结论立刻反转。

### E4 — 成本敏感性

1x / 3x / 5x 成本下均 **0/12 通过**（1x 已无人通过，更高成本只会更差）。

## 结论

- 在 16 年、12 个跨资产、28 个 OOS fold 的前推检验下，双均线择时**没有可泛化的 edge**：
  它要么在单票上直接亏（AAPL/MSFT），要么产出正分但跑输持有（指数/大盘股）。
- `^GSPC` 的 fold 明细显示选中配置在 5/10、10/20、10/30、20/60、15/40 间反复横跳，没有
  稳定解；很多 fold 仅 0–1 笔交易——fold 级别仍是小样本。
- **框架本身工作正常**：升级后的固定评估器 + 基准相对的严格门槛，把早期口径会放行的
  12 个假阳性全部挡下。这就是「固定评估器 + 反过拟合纪律」该有的样子——给出不舒服但
  正确的结论，而不是被前视偏差 + 样本内挑参美化的数字。

> 本仓库是方法论演示，不构成投资建议。要在此之上找到真实 edge，下一步应是 HL 循环里
> **提出有经济含义的结构性假设**（如趋势过滤、波动率定仓、多因子）并继续用本协议证伪，
> 而不是在双均线窗口上继续调参。
