# 常见陷阱与排障指南

本文记录项目中反复踩过的坑及标准解法，避免同一问题在后续会话中重演。这是「**持续学习**」能力的承载文件——每遇到一个非显然的坑（环境、依赖、数据、隐式约定、踩过的反模式），就把根因与标准解法补进来。

## 写法约定

- 一个坑一节，标题写**现象**，正文写**根因**与**标准解法**。
- 优先写「为什么会踩」而不只是「怎么绕过」，让后来者理解而非机械照搬。
- 已被回归测试 / golden case 固化的坑，注明对应测试位置，便于回放防遗忘。

## 模板

```markdown
## N. <一句话现象>

**现象**：……

**根因**：……

**标准解法**：……

**已固化**：（可选）对应回归测试 / golden case 路径
```

## 1. pandas pickle 缓存跨版本不可读

**现象**：复用 `example/.cache/*.pkl` 时，默认 Python / pandas 可能报
`NotImplementedError: (<StringDtype(...)>, array([...]))`，导致回测还没开始就失败。

**根因**：`pd.to_pickle` 生成的缓存不是稳定的跨版本数据格式。不同 Python / pandas
组合在 dtype 反序列化上可能不兼容，尤其是用旧环境写缓存、再用新环境读取时。

**标准解法**：优先在同一虚拟环境中读写缓存；若环境已变化，删除对应 `.cache/*.pkl`
并用聚宽凭证重新拉取数据。不要为了绕过缓存错误去修改固定评估器或基线口径。
