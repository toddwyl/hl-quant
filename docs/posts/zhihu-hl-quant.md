## 方法论的两个源头

这套做法不是凭经验拍出来的，而是把两份工作合流，落到「迭代交易策略」这件具体事上。

**其一，Andrej Karpathy 的 `auto-research`：固定评估器 + 单一可编辑程序。** 它的核心约束极简却极强——**评估流程必须钉死，优化过程只允许修改「被优化的那个程序program.md」本身**。

**其二，Jiayi Weng 的 _Learning Beyond Gradients_（启发式学习）。** 它的洞察不是「手写规则优于模型」，而是：当一个 agent 能够持续吸收失败、修改代码、跑实验、回放历史、压缩冗余规则时，启发式规则就不再是一次性补丁，而能被维护成一个**持续进化的系统**。通过可观测的途径，记录失败的历史，持续进化系统。

我做的事，就是把这两点焊在一起。

## 架构：把量化研究压成「一个策略 + 一个打分器」

整套系统的骨架可以缩成一张图：

```
strategy.py（唯一可编辑）──改它──▶ backtest.py（固定评估器）──▶ 一个分数
```

- **`strategy.py`**：策略的唯一语义源。整个迭代过程**只允许修改这一个文件**——提假设、改逻辑、调参数，都在这里发生。
- **`backtest.py`**：固定的评估口径。它负责拉数据、模拟成交、计入成本、计算指标，最后输出**一个标量分数**。一旦固定，就绝不允许为了「提分」去改动它。

为什么「评估器不许动」是整套方法的命门？因为一旦评估器可改，你就能在策略并没有真正变好的前提下让分数变好——这恰恰是大量量化实践「回测漂亮、实盘失效」的根源。把评估器钉死，「这一版到底有没有更好」才成为一个**可排序的客观问题**。

评分指标用 **Sortino ratio**——它只惩罚下行波动，更贴近「上行波动不是风险、下行波动才是风险」的交易直觉。指标可以按项目换，但「固定且唯一」这一点不变。

## 迭代循环：每一轮都是一次有纪律的小实验

每一轮迭代走同一套八步流程：

```
Probe（跑基线）→ Diagnose（诊断弱点）→ Propose（提一个有经济含义的假设）
→ Patch（只改 strategy.py）→ Evaluate（固定评估器打分）
→ Replay（回放，确认旧能力没被改坏）→ Decide（严格更好才留）→ Compress（压缩规则）
```

两条贯穿始终的硬规矩：

1. **只改策略文件**，评估器、数据源、成本模型、评分公式一律不动。
2. **每轮只动一个有经济含义的变量**，且必须严格更好才保留。

每一轮（无论成功失败）都写进账本，带上可解释的失败归因，供后续迭代复用——这正是「持续进化系统」落地的地方，而不是每次从零开始调参。

## 【核心】怎么真正防住过拟合：双重确认 + 市场规律闸

这是整套方法里最关键的一节。量化里绝大多数「看起来很神」的策略，死因都是同一个：**在历史数据上把参数雕到极致，雕的其实是噪声，不是规律。** 我用四层机制来压制它。

**第一层，留出一个独立的验证集。**
从回测样本里**随机抽取约三分之一作为验证集**（用固定的随机种子保证可复现），其余作为训练集。两者分工严格、互不越界：

- **训练集**：允许用于搜索、调参。
- **验证集**：**只用于检验泛化，不允许用来指导搜索方向**。它的作用是「否决」，不是「导航」。

**第二层，也是合格判据的核心：一个候选，只有在训练集和验证集上同时不退化（理想是同时提升），才算一次合格的策略改进。**
训练集大涨、验证集走平甚至下跌的候选，无论训练分数多好看，一律判定为过拟合并拒绝。训练与验证的分差超过阈值，即进入「拒绝区」。

**第三层，每个改动都必须符合市场规律。**
光有数据上的双重确认还不够——这一层是专门用来约束 agent 的。agent 很擅长在历史里挖出「能提分、却没有任何市场解释」的伪规律，而那几乎总是过拟合。所以我要求它提出的每一个改动，都能用一句话讲清楚背后的市场含义；讲不清的，分数再好也拒绝。

举一个我自己踩过的真实反面例子。某轮 agent 为了提分，提出了这样一条入场过滤规则：

```
跳过「连涨 2 天」的标的，但「连涨 3 天」「连涨 4 天」放行，到「连涨 5 天」又跳过
```

它在回测上确实把分数顶了上去，但你根本没法解释：凭什么连涨 2 天危险、连涨 3 天和 4 天反而安全、到 5 天又危险？这种**在离散变量上挖出的、碎片化又非单调的死区**（落到代码里就是 `跳过天数 = {2, 5}` 这种东一榔头西一棒子的集合），本质和「止损偏偏设成 8.2%」是同一类错误——拟合的是这段历史的噪声，不是任何可复用的市场结构。真正合格的改动应该是**单调、连续、可解释**的，比如「连涨超过 N 天后追高风险单调上升，统一回避」，而不是带着空洞的集合去迎合回测曲线。

**第四层，样本量门槛。**
有效交易数低于下限的候选直接判无效。靠一两笔幸运交易堆出来的高分，是运气不是 edge。

一句话概括这一节：**这套方法优化的不是回测分数，而是「不被回测骗到」。**

## 一个你能亲手复现的最小 demo

开源仓库里附了一个**玩具级**例子——上证指数上的双均线策略。它的价值不在「能赚钱」，而在**完整演示上面这套纪律如何运作**。

第 1 轮基线是 5/10 双均线：Sortino 仅 0.75，12 笔交易、胜率 41.67%，在震荡中频繁翻转、过早离场。
诊断后提出一个有经济含义的假设——把均线放慢到 10/20，以过滤日间噪声、把趋势持有得更久（这是灵敏度的连续调整，不是窄坑）。只改这两个参数，重跑固定评估器：

```
Sortino 0.75 → 2.06，总收益 +5.46% → +17.33%，胜率 41.67% → 57.14%，回撤同步收窄
```

但真正体现纪律的是下一步：我顺手试了 10/30、20/60，**分数更高**（2.26、2.09），可它们全年只剩 **2 笔 / 1 笔**交易——那是撞上了这段行情，不是稳健的 edge。**所以我拒绝了分数更高的候选，接受了分数次高、但样本量健康的 10/20。** 主动放弃更高分，正是这套方法的精髓。

## 从 demo 到实盘：同一套纪律，私有策略

文章开头那一个月 +16% 的实盘，用的不是这个公开的玩具双均线，而是我用**同一套纪律**——固定评估器、单一可编辑程序、每轮一个有经济含义的假设、训练/验证双重确认、样本量与窄坑两道闸——迭代出来的私有策略，细节不公开。

可公开、也最值得公开的，是方法本身。因为这套方法真正的价值不在某个具体策略，而在那条让你「少骗自己」的纪律；它对任何「有固定评估口径、追求可度量更优解」的问题都成立。

## 开源 & 一键装进你的 agent

整套循环我打包成了一个 Agent Skill，可直接装进你自己的 Claude Code / Cursor：

```bash
npx skills add toddwyl/hl-quant
```

装好后，跟 agent 说「用启发式探索优化这个策略」，它会按 `Probe → Diagnose → Propose → Patch → Evaluate → Replay → Decide → Compress` 的纪律帮你迭代：只改策略文件、严格门槛、反过拟合。

GitHub 仓库（含完整设计文档与可运行 demo）：**`github.com/toddwyl/hl-quant`**，觉得有用点个 Star。想深入聊启发式学习、量化策略与 Agent 工作流的，仓库里有微信交流群二维码。
