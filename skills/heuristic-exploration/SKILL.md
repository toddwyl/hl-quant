# 启发式探索：用固定评估器迭代优化量化策略

把策略研究压成两件东西，让「这版到底有没有更好」变成一个可排序的客观问题：

```
   HL →  strategy.py（唯一可编辑程序）──改它──▶  backtest.py（固定评估器）──▶ 一个分数
```

- **`strategy.py`** —— 策略唯一语义源。HL **只允许改这一个文件**。
- **`backtest.py`** —— 固定评估口径：同一段数据、同一评分公式、同一成本模型。一旦固定就**绝不修改**——否则「改了评估器把分数刷上去」，候选之间不再可比。

参考实现：本仓库 [`example/strategy.py`](../../example/strategy.py) + [`example/backtest.py`](../../example/backtest.py)。

## 核心循环（每轮迭代）

```
Probe → Diagnose → Propose → Patch → Evaluate → Decide → Compress
```

1. **Probe（探测）**：跑固定评估器，拿到基线分数和各项指标。
2. **Diagnose（诊断）**：看它**为什么差**——交易太多还是太少？在哪类行情亏？是过早离场、追高、还是被噪声甩出？结论要可解释，别只盯着一个分数。
3. **Propose（假设）**：提**一个**有经济含义的假设。一次只动一个变量，便于归因。
4. **Patch（修改）**：**只改 `strategy.py`**（逻辑或参数）。评估器、数据源、评分公式一律不动。
5. **Evaluate（评估）**：重跑固定评估器，对比分数与全部关键指标。
6. **Decide（决策）**：过下面的严格门槛才接受，否则丢弃这条假设、回到 Diagnose。
7. **Compress（压缩）**：移除零触发 / 负贡献的规则，保留更小、更可解释的结构，不无限叠 if-else。

## 接受门槛（严格）

候选必须**在分数上严格高于基线**，且关键指标不出现明显退化：

| 指标 | 要求 |
|------|------|
| **score**（主决策指标） | 严格高于基线 |
| 总收益 / 年化 / Sharpe | 不低于基线（容忍极小噪声波动） |
| 最大回撤 | 不高于基线 |
| 胜率 | 不明显下降 |
| 有效样本量（交易笔数） | 足够多，单边 1~2 笔的「胜率 100%」不算数 |

分数上去了但某个关键指标塌了，或交易笔数太少 → **拒绝**。没有部分接受。

## 反过拟合纪律（强制）

1. **连续变量上禁止挖窄坑**：在 MFI / RSI / 量比 / 均线窗口等连续维度上挖 1~2 点宽的离散死区，是过拟合的典型标志。合理的过滤必须 (a) 是有明确经济含义的连续区间，或 (b) 是规则级别的开关。
2. **每个改动都要有经济解释**：说得清「为什么这样改在市场上更合理」，而不是「调到这个数分数最高」。
3. **有效样本量**：交易笔数太少（如 1~2 笔）时的高分更可能是撞上了这段行情，不是稳健规律——宁可要「分数全面更好 + 笔数健康」的候选，也不追绝对最高分。
4. **禁止未来信息**：任何决策只能用「决策时点已知」的信息。信号用已完成的 K 线，避免用当日未完成的最高/最低/振幅来过滤买入。

## 禁止修改

为了「提高分数」，**禁止**改下面任何一项（否则分数不可比）：

- 评估器脚本本身（`backtest.py`）
- 数据源、回测区间、评分公式、成本模型、有效性门槛

如果评估基础设施本身坏了（认证失败、数据缺口），**停下来报告**，不要靠改基准绕过问题。

## 留痕

每轮（含失败）都记录：候选 diff / 参数、score、总收益、年化、Sharpe、最大回撤、胜率、交易笔数，以及一句策略想法说明。失败也要留可解释的归因，供跨轮次学习。
